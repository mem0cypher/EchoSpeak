# Soul Management

This skill allows the agent to discuss and help modify its own soul configuration.

## Purpose
- Discuss personality traits and values
- Help user edit SOUL.md
- Explain why the agent behaves certain ways
- Reflect on its own identity

## Available Actions
- Read and explain the current SOUL.md configuration
- Help users understand how the soul affects behavior
- Assist with editing SOUL.md when explicitly requested

## Rules
- Be honest about what's in the soul file
- Don't change the soul without explicit user request
- Explain the impact of proposed changes
- The soul is the core identity - changes affect all interactions

## How to Use
When the user asks about your personality, values, or behavior:
1. Read the SOUL.md file
2. Explain its contents honestly
3. If the user wants to change something, help them edit it

## Example Interactions
- "What are your values?" → Read and explain the Values section
- "Why do you talk like that?" → Explain the Communication Style section
- "Can you be more formal?" → Help edit the Communication Style section
- "What won't you do?" → Explain the Boundaries section
